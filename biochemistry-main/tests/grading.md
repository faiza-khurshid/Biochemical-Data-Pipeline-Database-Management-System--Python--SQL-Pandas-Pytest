# Grading Criteria

1. gitignore exists and added file to exclude added 1P
2. write a README.md file describing the project 1P
3. write docstrings for all functions and classes 2P
4. use typings everywhere 2P
5. create issues for all tasks (be careful, tip: create subitems in issues to reduce the number of issues) 1P
6. create for each issue a branch (create branch one after the other, implement the code in the branch, push to the branch and create a merge request on GitLab) 3P
7. Merge on GitLab and delete source branch (by default activated) 2P
8. create test ipyn notebook (`notebooks/demo.ipynb`) to test the functionality of your library 3P
9.  All tests are passing 10P
10. Extra points for normalization of substrates and products 2P

In sum, you can get a maximum of 25 (+2 extra) points. The grading will be done based on the main branch and the code you implemented according to the tests.
